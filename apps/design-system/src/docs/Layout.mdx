# Layout & Page Structure

All AtomyQ screens follow a predictable shell structure so users never get lost between modules.

---

## Global Layout

```
┌──────────────────────────────────────────────────────┐
│ Sidebar (240px / 64px collapsed)  │ Header (56px)    │
│                                   │──────────────────│
│ • Logo                            │ Content Area     │
│ • Nav sections                    │                  │
│   – Workspace                     │ • Page title     │
│   – Collaboration                 │ • Breadcrumbs    │
│   – Governance                    │ • Actions        │
│   – Administration                │ • Main content   │
│                                   │                  │
│ • Collapse toggle                 │──────────────────│
│                                   │ Footer (32px)    │
└──────────────────────────────────────────────────────┘
```

## Grid System

- **12-column grid** with 16px gutters
- Content max-width: **1440px** (2xl breakpoint)
- Side padding: 24px (desktop), 16px (tablet)

## Responsive Breakpoints

| Breakpoint | Width | Behaviour |
|---|---|---|
| `sm` | 640px | Mobile — single column, no sidebar |
| `md` | 768px | Tablet — sidebar collapses to icons |
| `lg` | 1024px | Small laptop — sidebar can be toggled |
| `xl` | 1280px | Laptop — comfortable layout |
| `2xl` | 1440px | **Primary design target** |
| `3xl` | 1920px | Wide desktop — extra table column space |

## Page Layout Templates

### Dashboard Layout
KPI cards in a 4-column grid at top, followed by 2-column main/side layout below.

```
┌─────┬─────┬─────┬─────┐
│ KPI │ KPI │ KPI │ KPI │
├───────────┬─────────────┤
│ Chart /   │ Tasks /     │
│ Table     │ Alerts      │
│ (8 cols)  │ (4 cols)    │
└───────────┴─────────────┘
```

### Data Table Layout
Full-width table with filter bar above and pagination below.

```
┌─────────────────────────┐
│ Filter bar + Actions    │
├─────────────────────────┤
│ Data Table (12 cols)    │
│ • Sortable headers      │
│ • Row selection          │
│ • Inline status badges   │
├─────────────────────────┤
│ Pagination              │
└─────────────────────────┘
```

### Form Layout
Two-column form with sidebar summary.

```
┌───────────┬─────────────┐
│ Form      │ Summary     │
│ Fields    │ Panel       │
│ (8 cols)  │ (4 cols)    │
├───────────┴─────────────┤
│ Form actions (right)    │
└─────────────────────────┘
```

### Detail View Layout
Header section with entity info, tabbed content below.

```
┌─────────────────────────┐
│ Entity Header + Actions │
├─────────────────────────┤
│ Tab Bar                 │
├─────────────────────────┤
│ Tab Content             │
└─────────────────────────┘
```

### Multi-Panel Layout (Comparison)
Side-by-side panels for comparing vendor quotes.

```
┌───────┬───────┬───────┐
│ Item  │Vendor │Vendor │
│ List  │  A    │  B    │
│(3col) │(4col) │(4col) │

```

## Spacing System

All spacing uses a **4px base unit**:

| Token | Value | Usage |
|---|---|---|
| `space-1` | 4px | Icon gaps, tight inline spacing |
| `space-2` | 8px | Table cell padding, compact gaps |
| `space-3` | 12px | Form field padding |
| `space-4` | 16px | Default content padding |
| `space-6` | 24px | Card body padding |
| `space-8` | 32px | Section separation |
| `space-12` | 48px | Major layout blocks |

## Navigation Patterns

- **Sidebar**: Always visible (collapsible on tablet). Organised into 4 sections.
- **Breadcrumbs**: Shown in header for all pages deeper than root.
- **Tabs**: Used within detail views to organise related content.
- **Slide-over Modals**: For contextual actions (25-30% screen width, from the right).
- **Full Modals**: For focused workflows requiring user's full attention.
