# Performance Guidelines

ERP users work long hours. Slow UI is not just an annoyance — it's a productivity drain. These guidelines ensure AtomyQ stays responsive.

---

## Core Metrics

| Metric | Target | Why |
|---|---|---|
| First Contentful Paint | < 1.5s | Users see content quickly |
| Largest Contentful Paint | < 2.5s | Main content renders fast |
| Interaction to Next Paint | < 200ms | UI responds instantly |
| Bundle size (initial) | < 200KB gzipped | Fast initial load |
| API response time | < 500ms (p95) | Smooth data operations |

```tsx
  const parentRef = useRef<HTMLDivElement>(null);
  const virtualizer = useVirtualizer({
    count: rows.length,
    getScrollElement: () => parentRef.current,
    estimateSize: () => 40, // row height
  });
  // render only visible rows...
}
```

## Debounced Search

All search inputs should debounce API calls by **300ms**:

```tsx
function useDebounce<T>(value: T, delay = 300): T {
  const [debounced, setDebounced] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(timer);
  }, [value, delay]);

  return debounced;
}
```

## API Caching Strategy

| Data Type | Cache Duration | Strategy |
|---|---|---|
| User permissions | 15 minutes | Cache in memory, refresh on focus |
| RFQ list | 30 seconds | Stale-while-revalidate |
| Vendor list | 5 minutes | Cache with background refresh |
| Comparison results | 1 minute | Invalidate on new comparison run |
| Static config | 1 hour | Cache with manual refresh |

```tsx
// Example with React Query / TanStack Query
const { data } = useQuery({
  queryKey: ['rfqs', filters],
  queryFn: () => fetchRFQs(filters),
  staleTime: 30_000,       // 30 seconds
  gcTime: 5 * 60_000,      // 5 minutes garbage collection
  refetchOnWindowFocus: true,
});
```

## Rendering Optimisation

- **Memoize expensive computations**: Use `useMemo` for derived data (sorted/filtered lists)
- **Memoize callbacks**: Use `useCallback` for handlers passed to child components
- **Avoid inline objects**: Don't create new objects/arrays in render body
- **Use `React.memo`**: For list item components rendered 50+ times

```tsx
// Memoize sorted/filtered table data
const sortedData = useMemo(
  () => data.toSorted((a, b) => /* sort logic */),
  [data, sortColumn, sortDirection]
);
```

## Image & Asset Guidelines

- Use **WebP** format for all images
- Provide **width** and **height** attributes to prevent layout shift
- Lazy load images below the fold: `loading="lazy"`
- SVG icons inline (not as images) for crispness at all sizes
