# Accessibility Standards

AtomyQ targets **WCAG 2.1 Level AA** compliance. ERP users work 8+ hours daily — accessibility is not optional, it's a productivity requirement.

---

## Keyboard Navigation

### Global Shortcuts

| Shortcut | Action |
|---|---|
| `/` | Focus global search |
| `Ctrl+K` | Command palette |
| `Ctrl+Enter` | Submit current form |
| `Escape` | Close modal / popover |
| `Tab` | Navigate forward |
| `Shift+Tab` | Navigate backward |
| `Enter` / `Space` | Activate focused element |
| `Arrow keys` | Navigate within menus, tabs, radio groups |

### Focus Management

- All interactive elements must have a visible focus indicator (2px ring, brand color)
- Focus must be trapped inside modals when open
- When a modal closes, focus returns to the triggering element
- Skip-to-content link should be the first focusable element

```tsx
// Focus ring utility
className="outline-none focus-visible:ring-2 focus-visible:ring-[var(--aq-brand-500)]"
```

## Screen Reader Support

### Semantic HTML

Always use semantic elements:

| Element | Usage |
|---|---|
| `<nav>` | Navigation regions |
| `<main>` | Primary content area |
| `<aside>` | Sidebar, supplementary content |
| `<table>` | Data tables (not `<div>` grids) |
| `<form>` | Form containers |
| `<button>` | Interactive actions (never `<div onClick>`) |

### ARIA Attributes

| Attribute | Usage |
|---|---|
| `aria-label` | Icon-only buttons, unlabelled controls |
| `aria-describedby` | Link inputs to error/hint messages |
| `aria-invalid` | Indicate validation errors |
| `aria-current="page"` | Active navigation item |
| `aria-sort` | Sortable table columns |
| `aria-busy` | Loading states |
| `aria-live="polite"` | Dynamic content updates (toasts, alerts) |
| `role="alert"` | Error messages, critical notifications |
| `role="status"` | Loading spinners, progress indicators |

## Colour Contrast

### Minimum Ratios

| Context | Ratio | Standard |
|---|---|---|
| Normal text (< 18px) | **4.5:1** | WCAG AA |
| Large text (≥ 18px or 14px bold) | **3:1** | WCAG AA |
| UI components & icons | **3:1** | WCAG AA |
| Focus indicators | **3:1** | WCAG AA |

### Rules

- **Never communicate with colour alone**: Always pair colour with text, icons, or patterns
- Status badges include both a coloured dot AND text label
- Error states use red border AND error icon AND error text
- Chart data uses colour AND pattern/label differentiation

## Motion & Animation

- All animations should complete in **under 200ms** for interactions, **under 300ms** for transitions
- Respect `prefers-reduced-motion`:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Form Accessibility Checklist

- [ ] Every input has a visible `<label>` (or `aria-label` for icon inputs)
- [ ] Required fields indicated with `*` AND `required` attribute
- [ ] Error messages linked via `aria-describedby`
- [ ] Validation errors announced with `role="alert"`
- [ ] Form groups use `<fieldset>` and `<legend>`
- [ ] Submit buttons are `<button type="submit">`
- [ ] Disabled state uses both `disabled` attribute AND visual indicator

## Testing Checklist

- [ ] Tab through entire page — all interactive elements reachable
- [ ] Activate every control with keyboard (Enter/Space)
- [ ] Test with screen reader (NVDA/VoiceOver)
- [ ] Verify focus visible on all interactive elements
- [ ] Check colour contrast with automated tool
- [ ] Verify all images have alt text
- [ ] Ensure modals trap focus correctly
- [ ] Test with browser zoom at 200%
