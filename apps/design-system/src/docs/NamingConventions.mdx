# Naming Conventions

Consistent naming is critical in a large ERP system with 50+ modules. These conventions ensure predictability for all team members.

---

## Component Names

All design system components use the `AtomyQ` prefix:

| Component | Name |
|---|---|
| Button | `AtomyQButton` |
| Table | `AtomyQTable` |
| Modal | `AtomyQModal` |
| Input | `AtomyQInput` |
| Badge | `AtomyQBadge` |
| Sidebar | `AtomyQSidebar` |
| Toast | `AtomyQToast` |
| KPI Card | `AtomyQKPICard` |

### File Naming

```
src/components/basic/AtomyQButton.tsx       // Component
src/components/basic/AtomyQButton.stories.tsx // Stories
src/components/basic/AtomyQButton.test.tsx   // Tests
```

## Permission Keys

Follow `resource.action` pattern in lowercase:

```typescript
// Resource permissions
'rfq.create'          // Create a new RFQ
'rfq.read'            // View RFQ list and details
'rfq.update'          // Edit RFQ metadata
'rfq.delete'          // Delete/archive an RFQ

'comparison.read'     // View comparison results
'comparison.run'      // Execute a new comparison
'comparison.export'   // Export comparison data

'quotation.approve'   // Approve a vendor quote
'quotation.reject'    // Reject a vendor quote
'quotation.escalate'  // Escalate for higher approval

'vendor.create'       // Add a new vendor
'vendor.read'         // View vendor list
'vendor.update'       // Edit vendor details
'vendor.invite'       // Send vendor invitations

'admin.users'         // Manage users
'admin.roles'         // Manage roles
'admin.settings'      // System settings
'admin.all'           // Full admin access
```

## API Endpoints

```
GET    /api/v1/rfqs                    # List RFQs
POST   /api/v1/rfqs                    # Create RFQ
GET    /api/v1/rfqs/{id}               # Get RFQ detail
PUT    /api/v1/rfqs/{id}               # Update RFQ
DELETE /api/v1/rfqs/{id}               # Delete RFQ

GET    /api/v1/rfqs/{id}/quotes        # List quotes for RFQ
POST   /api/v1/rfqs/{id}/comparisons   # Run comparison
GET    /api/v1/comparisons/{id}        # Get comparison result

GET    /api/v1/vendors                 # List vendors
POST   /api/v1/vendors/{id}/invite     # Invite vendor to RFQ
```

## CSS Variable Naming

Design tokens use the `--aq-` prefix:

```css
/* Backgrounds */
--aq-bg-canvas
--aq-bg-surface
--aq-bg-elevated

/* Text */
--aq-text-primary
--aq-text-secondary
--aq-text-muted

/* Brand colours */
--aq-brand-600
--aq-success-500
--aq-danger-500

/* Spacing */
--aq-space-4

/* Shadows */
--aq-shadow-md
```

## ID Formats

| Entity | Format | Example |
|---|---|---|
| RFQ | `RFQ-YYYY-NNN` | `RFQ-2026-001` |
| Quote Submission | `QS-NNN` | `QS-001` |
| Comparison | `CMP-NNN` | `CMP-001` |
| Vendor | `V-NNN` | `V001` |
| User | `U-NNN` | `U001` |
| Approval | `APR-NNN` | `APR-001` |
| Task | `T-N` | `T1` |

## Event & Action Naming

```typescript
// Handler props: on + Action
onSort, onFilter, onPageChange, onRowClick, onSelectRow

// State: is + Adjective
isLoading, isOpen, isSelected, isDisabled

// Boolean props: has/is/can/should
hasPermission, isReadOnly, canEdit, shouldAutoSave
```
